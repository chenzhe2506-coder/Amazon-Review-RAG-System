# 🤖 03. RAG 智能客服引擎 (Retrieval-Augmented Generation)

## 🎯 模块简介
本模块是系统的“终极合体形态”。它将检索系统（Retriever）与大模型生成（Generator）完美缝合，构建了一个具备“自主翻阅公司规定并据此回复用户”的初级智能体（Agent）。

## 🛠️ 核心工作流
1. **意图接收**: 捕获前端或 RPA 抓取到的用户复杂长文本。
2. **记忆检索 (RAG)**: 触发 ChromaDB，根据用户文本搜索最符合的内部业务 SOP。
3. **图纸组装 (Prompting)**: 将搜索到的硬性 SOP 规则与用户的原话放入预设的 Prompt 模板中。
4. **高情商生成 (Generation)**: 限制大模型绝对服从 SOP 规则，输出带有同理心、格式规范、且提供实质性解决方案的标准客服邮件。

## 🚀 运行前提
必须同时满足模块 01 与 02 的环境配置（包含 `.env` 密钥加载与 BGE 模型就绪）。

## 💡 业务价值
* 彻底消除大模型“胡说八道（幻觉）”的风险，保证客服回复 100% 合规。
* 极大降低人工处理低级且重复客诉的时间成本。
* 可直接通过 API 对接影刀 RPA 或网页端，实现自动巡检并回复。
